允许合并另一个构造者的选项

mergeOptions 函数中

Vue 构造函数拥有 options 属性，Vue.extend 创建的子类也拥有 options 属性,所以在合并 options 时允许合并另一个构造者的选项

规范化 props

mergeOptions 函数中

这三个函数用来规范选项的，例如 props 可以写成字符串数组，可以写成对象(规范写法)，进行合并时需要统一处理

normalizeProps(child, vm);

props 如果是字符串数组，就遍历数组，并判断当前项是否为字符串，将名称中的中划线转换为驼峰(camelize 方法)，res[name] = { type: null }

props 如果为对象，先判断是否为对象(isPlainObject 方法)，遍历对象，拿到当前项的值(val)和转换后的名称(name)，如果当前项的值为对象，res[name] = val，否则res[name] = { type: null }

props 如果即不是数组又不是纯对象，在非生产环境就抛出警告

最后用res替换options.props

规范化 inject

normalizeInject(child, vm);

使用 inject 缓存 options.inject，如果 inject 不存在，直接 return。声明const normalized = options.inject = {}options.inject 和 normalized 定义为空对象，都是同一个引用，对 normalized 的修改也会影响 options.inject

inject 选项应该是一个数组或对象(规范写法)，对 inject 的规范化和 props 类似

如果 inject 是数组，遍历数组，将当前值(inject[i])作为 key，将{ from: inject[i] }作为值，修改 normalized(normalized[inject[i]] = { from: inject[i] })

如果 inject 是对象，遍历对象，如果当前值(inject[key])是对象，进行混合(extend({ from: key }, inject[key]))，如果不是对象，直接将inject[key]作为form的值({ from: inject[key] })

inject 如果即不是数组又不是纯对象，在非生产环境就抛出警告

规范化 directives

normalizeDirectives(child)

directives 可以直接写成函数或对象(包含 bind 方法),包含 bind 的是规范写法
拿到 options.directives 进行判断，如果存在，遍历对象，判断当前项 def(dirs[key])是否是函数，是则将{ bind: def, update: def }作为 def 的值

三个规范函数后

这是对 extends 和 mixins 进行处理，extendsFrom保存对child.extends的引用，如果extendsFrom存在就合并mergeOptions(parent, extendsFrom, vm)；如果child.mixins存在，遍历数组，合并 parent、数组元素和 vm
