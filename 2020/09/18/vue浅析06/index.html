数据响应系统

在 initState 函数中使用initProps初始化props属性，使用initMethods 初始化methods，使用initData初始化data，使用initComputed初始化computed，使用initWatch初始化watch。

initState 中初始化 data：

opts.data存在则初始化data，否则通过observe观察一个空对象，$data属性是一个访问属性，其代理值为_data。

initData 函数

initData 在 core/instance/state.js 文件中

定义data变量引用vm.$options.data，而vm.$options.data会被处理成函数，该函数执行结果才是需要的数据。使用typeof判断类型的原因，是因为beforeCreate生命周期是在mergeOptions 函数之后initData 之前调用的，可以在该周期中修改vm.$options.data。如果vm.$options.data为函数，通过getData拿到数据。

通过getData拿到数据后，赋值给vm._data，重写data变量，此时data是最终数据对象。

通过isPlainObject判断data是否是一个纯对象，不是在非生产环境，打印警告信息。

遍历 data 的键，看在 methods 和 props 中有没有相同的键，如果相同在非生产环境，打印警告信息。因为定义在 data、methods、props 中的数据可以通过实例对象代理访问，有相同的键时，会出现覆盖现象。

isReserved判断data中的键(key)是否是以$或_开头的，Vue自身的属性是以$或_开头的，不是以$或_开头的，执行proxy实现实例对象的代理访问。

proxy 通过Object.defineProperty函数在实例对象vm上定义与data数据字段同名的访问器属性，并且这些属性代理的值是vm._data上对应属性的值。

最后调用调用observe函数将data数据对象转换成响应式的。

  • 根据 vm.$options.data 选项获取真正想要的数据（注意：此时 vm.$options.data 是函数）
  • 校验得到的数据是否是一个纯对象
  • 检查数据对象 data 上的键是否与 props 对象上的键冲突
  • 检查 methods 对象上的键是否与 data 对象上的键冲突
  • 在 Vue 实例对象上添加代理访问数据对象的同名属性
  • 最后调用 observe 函数开启响应式之路
