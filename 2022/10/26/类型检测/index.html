类型检测

发表于 2022-10-26

类型检测

  • typeof
    • 校验基本类型
    • typeof NaN === ‘number’
    • typeof 普通对象/数组对象/正则对象/日期对象都是 object
  • instanceof
    • 检测引用类型
    • 检测当前实例是否属于这个类
    • 机制：只要当前类出现在实例的原型上都是 true
  • constructor
    • 支持基础类型
    • constructor 可以更改，不准
  • Object.prototype.toString.call()
    • [object xxx]

手写instanceof

类型转换

原始值 转换目标 结果
number 布尔值 除了0、-0、NaN都是true
string 布尔值 除了空串都为true
undefined、null 布尔值 false
引用类型 布尔值 true
number 字符串 5 => ‘5’
boolean、函数、Symbol 字符串 ‘true’
数组 字符串 [1,2] => ‘1,2’
对象 字符串 ‘[object Object]’
string 数字 ‘1’ => 1 ‘a’ => NaN
数组 数字 空数组为0，存在一个元素并且为数字转数字，其它情况NaN
null 数字 0
除了数组的引用类型 数字 NaN
Symbol 数字 抛错

object的转换规则

  • 优先调用 Symbol.toPromitive 方法
  • 调用 valueOf()，如果转换为基础类型，则返回
  • 调用 toString()，如果转换为基础类型，则返回
  • 如果都没有返回基础类型，会报错

    ==隐式类型转换

  • 如果类型相同，无须进行类型转换
  • 如果其中一个是 null 或者 undefinded，另一个操作符必须为 null 或 undefinded，才会返回 true，否则都返回 false
  • 如果其中一个是 Symbol 类型，返回 false
  • 两个操作值为 string number，将字符串转换为 number
  • 如果一个为 boolean，那么转换为 number
  • 如果一个为 object，另一个为string、number或者Symbol，把 object转换为原始类型再判断
